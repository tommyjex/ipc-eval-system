# 项目上下文

## 目的

摄像头场景的大模型应用效果评测平台，提供数据集管理、标注、模型选择、评测、打分、报告生成等核心功能，帮助用户系统化评估大模型在摄像头场景中的应用效果。

## 技术栈

- 前端：TypeScript, React
- 后端： Python
- 数据库：MySQL
- 存储：对象存储（用于图片/视频数据）

## 项目约定

### 代码风格

- 使用中文编写文档、注释和提交信息
- 遵循 ESLint / Prettier 代码格式化规范
- 组件命名采用 PascalCase，函数/变量采用 camelCase
- 文件命名采用 kebab-case

### 架构模式

- 前后端分离架构
- RESTful API 设计
- 模块化组件设计
- 分层架构：表示层、业务层、数据层

### 测试策略

- 单元测试覆盖核心业务逻辑
- 集成测试覆盖 API 端点
- E2E 测试覆盖关键用户流程

### Git工作流

- 主分支：main（生产环境）
- 开发分支：develop
- 功能分支：feature/\*
- 修复分支：bugfix/\*
- 提交信息使用中文，格式：`类型: 简短描述`

## 领域上下文

- **摄像头场景**：监控、安防、交通、零售等场景下的摄像头图像/视频数据
- **大模型评测**：评估视觉语言模型（VLM）在特定场景下的理解、分析和推理能力
- **评测维度**：准确率、响应时间、鲁棒性、场景适应性等

## 重要约束

- 支持大规模图片/视频数据的存储和处理
- 评测过程需要可追溯、可复现
- 报告生成需要支持多种格式导出

## 外部依赖

- 大模型 API（OpenAI、Claude、通义千问等）
- 对象存储服务（OSS/S3）
- 数据库服务
