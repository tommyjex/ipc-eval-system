# 变更：支持任务 Prompt 模板管理与套用

## 为什么
当前评测任务的 `prompt` 只能在创建任务或编辑任务时手工输入，无法按业务场景复用已有提示词，也缺少统一维护入口，导致相同场景下的任务 Prompt 容易重复录入且内容不一致。

为了提高任务配置效率并保证同类场景下的 Prompt 一致性，系统需要支持可维护的 Prompt 模板，并在任务创建和任务详情编辑时按评测集业务场景选择模板直接套用。

## 变更内容
- 新增任务 Prompt 模板数据模型与 CRUD 接口
- 模板按业务场景分类，并复用评测集已有的 `scene` 分类
- 在创建任务时支持按评测集场景选择 Prompt 模板并覆盖任务 `prompt`
- 在编辑任务详情时支持按评测集场景选择 Prompt 模板并覆盖任务 `prompt`
- 提供最小可用的 Prompt 模板管理界面，支持创建、编辑、删除模板

## 影响
- 受影响规范：task-management（修改）
- 受影响代码：
  - 后端：Prompt 模板模型、接口、应用启动建表
  - 前端：任务创建页、任务详情编辑页、模板管理弹窗
