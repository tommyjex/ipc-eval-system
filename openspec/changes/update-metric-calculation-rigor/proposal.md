# 变更：收紧召回率与准确率计算口径

## 为什么
当前智能评分由大模型直接输出单条 `recall` 与 `accuracy` 分数，缺少对“评测单元”“命中规则”“空样本”“无效样本”“任务级聚合方式”的明确约束，导致结果可解释性、可复现性和跨任务可比性不足。

同时，任务列表页和对比页当前使用简单平均值汇总 `avg_recall` 与 `avg_accuracy`，会把“事件很多的复杂样本”和“事件很少的简单样本”赋予相同权重，容易放大样本分布差异带来的偏差。

## 变更内容
- 为单条评测结果定义统一的“可评测单元”抽取和标准化格式，先把 `ground_truth` 与 `model_output` 规范化到同一比较空间
- 将单条召回率与准确率从“模型直接给分”改为“模型抽取 + 系统规则匹配 + 公式计算”的两阶段方案
- 明确单条 `recall` 的计算公式为 `TP / (TP + FN)`，并将当前产品中的 `accuracy` 明确定义为“预测准确率”，统计口径等同于 `TP / (TP + FP)`；必要时增加 `precision` 别名字段以减少歧义
- 为任务级结果同时提供 `micro` 和 `macro` 两种聚合口径，并补充 `coverage_rate`、`unscorable_count`、`empty_sample_pass_rate` 等辅助指标
- 在前端结果页、任务列表页和对比页补充指标口径说明，避免用户将简单平均误解为全局真实表现

## 影响
- 受影响规范：`task-management`
- 受影响代码：
  - 后端：智能评分流程、结果标准化、匹配算法、任务级聚合查询
  - 后端：任务结果 Schema 与统计汇总字段
  - 前端：任务详情、任务列表、任务对比页面的指标展示与口径说明
  - 测试：单条评分、空样本、异常样本、任务聚合的单元与集成测试
