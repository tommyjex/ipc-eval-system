## 修改需求
### 需求：对评测结果执行智能评分
系统必须允许用户在任务详情页对已有评测结果执行智能评分，并为每条结果生成召回率、准确率和评分理由。

#### 场景：点击智能评分按钮
- **当** 用户在任务详情页点击“智能评分”按钮
- **那么** 系统启动当前任务下已有评测结果的批量智能评分流程

#### 场景：使用固定评分模型
- **当** 系统执行智能评分
- **那么** 系统必须使用固定评分模型完成结果标准化或辅助抽取

#### 场景：评分由系统公式计算
- **当** 系统完成单条结果评分
- **那么** 系统必须基于标准化后的 `ground_truth` 与 `model_output` 计算 `TP`、`FP`、`FN`
- **并且** 系统必须使用 `TP / (TP + FN)` 计算召回率
- **并且** 系统必须使用 `TP / (TP + FP)` 计算当前产品中的准确率
- **并且** 系统不得仅依赖评分模型直接返回最终分数作为唯一评分结果

#### 场景：结构化评分结果
- **当** 单条评测结果完成智能评分
- **那么** 系统保存并返回该条结果的 `召回率`、`准确率` 和 `评分理由`
- **并且** 系统保存或可重建该条结果的 `TP`、`FP`、`FN`、评分版本和命中摘要

#### 场景：空样本与不可评测样本
- **当** 单条结果不存在可评测单元或无法完成稳定解析
- **那么** 系统必须按“空样本”或“不可评测样本”分类处理
- **并且** 系统不得把不可评测样本静默计入主指标平均值

### 需求：任务级评分统计口径
系统必须以可解释、可比的方式汇总任务级召回率和准确率。

#### 场景：提供双重聚合口径
- **当** 系统汇总任务级评分结果
- **那么** 系统必须同时支持 `micro` 和 `macro` 两种聚合口径

#### 场景：提供覆盖与空样本统计
- **当** 用户查看任务级评分统计
- **那么** 系统必须提供 `coverage_rate`
- **并且** 系统必须提供 `empty_sample_pass_rate`
- **并且** 系统必须提供不可评测样本数量

#### 场景：默认展示可比口径
- **当** 用户在任务列表页、任务详情页或任务对比页查看汇总指标
- **那么** 系统默认展示的召回率和准确率必须采用统一的任务级聚合口径
- **并且** 系统必须向用户说明该口径不是简单样本均值

### 需求：在结果表展示结构化评分指标
系统必须在任务详情页结果表中展示召回率、准确率和评分理由。

#### 场景：显示结构化评分列
- **当** 用户查看任务详情页的评测结果表
- **那么** 系统展示 `召回率`、`准确率`、`评分理由` 三列

#### 场景：显示指标定义说明
- **当** 用户查看任务级或结果级评分指标
- **那么** 系统必须提供指标口径说明
- **并且** 系统必须明确当前“准确率”的统计定义
