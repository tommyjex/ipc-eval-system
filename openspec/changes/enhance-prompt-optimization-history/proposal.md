# 变更：增强提示词优化的历史版本与效果对比可读性

## 为什么
当前提示词优化闭环已经支持人工微调、创建对比任务、查看任务级指标对比和替换任务 Prompt，但还有两个明显缺口：

- 任务级指标对比虽然能展示“优化前/优化后”数值，但缺少“提升/下降”的直观视觉反馈，用户需要自己逐项比较
- 提示词优化结果目前按任务只有一条记录，重新生成会覆盖上一版，缺少历史版本追溯能力

这会降低提示词优化链路的可解释性和可追踪性，尤其是在多次迭代 Prompt 的场景下。

## 变更内容
- 为“优化前后效果对比”增加提升/下降的颜色和箭头反馈
- 为提示词优化结果增加历史版本能力，避免重新生成时覆盖上一版
- 支持查看任务下的历史优化记录，并切换查看某一版的分析、优化提示词和对比结果
- 新生成的优化结果默认创建为新版本，而不是覆盖旧版本

## 影响
- 受影响规范：`task-management`
- 受影响代码：
  - 后端：提示词优化数据模型、列表/详情接口、版本切换与版本引用
  - 前端：任务详情页优化结果区域、历史版本列表、指标对比视觉反馈
