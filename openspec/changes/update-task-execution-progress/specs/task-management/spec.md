## 修改需求
### 需求：在任务详情页编辑任务信息
系统必须在任务详情页展示任务运行所需的关键上下文和逐条结果状态，包括评测集名称、自定义标签以及每条数据的运行状态和评测结果。

#### 场景：任务详情页展示逐条运行状态
- **当** 用户打开一个已启动或已完成的评测任务详情页
- **那么** 系统展示任务内每条评测数据的运行状态

#### 场景：任务详情页展示逐条结果
- **当** 某条评测数据已经完成评测
- **那么** 系统展示该数据的模型输出、评测结果和完成时间

#### 场景：任务详情页展示失败原因
- **当** 某条评测数据评测失败
- **那么** 系统展示该数据的失败状态和失败原因

## 新增需求
### 需求：按任务级提示词和自定义标签执行评测
系统必须在运行评测任务时，使用任务级 Prompt 与评测集自定义标签组合后的提示词逐条调用任务指定的大模型。

#### 场景：任务 Prompt 优先
- **当** 任务配置了 Prompt 并启动运行
- **那么** 系统优先使用任务 Prompt 作为本次评测的基础提示词

#### 场景：拼接自定义标签
- **当** 任务所属评测集配置了自定义标签
- **那么** 系统将自定义标签拼接到本次评测提示词中，一并发送给模型

#### 场景：媒体处理策略
- **当** 系统处理不同类型的数据
- **那么** 图片直接传图、普通视频保持直传、GIF 继续采用抽帧方式评测

### 需求：逐条记录评测执行状态
系统必须为任务中的每条评测数据维护独立的执行状态，以便前端展示运行进度和结果。

#### 场景：任务启动后初始化逐条状态
- **当** 用户启动一个评测任务
- **那么** 系统为任务内每条数据初始化独立的执行状态记录

#### 场景：单条数据运行中
- **当** 系统开始处理某条评测数据
- **那么** 系统将该数据的执行状态更新为运行中

#### 场景：单条数据成功完成
- **当** 某条评测数据完成模型调用与结果写入
- **那么** 系统将该数据的执行状态更新为成功并记录完成时间

#### 场景：单条数据执行失败
- **当** 某条评测数据处理失败
- **那么** 系统将该数据的执行状态更新为失败并记录错误原因
