## 上下文
当前 `evaluation_tasks.status` 只能表达任务整体状态，`task_results` 只在单条数据评测成功后插入记录，失败时仅打印日志，前端无法感知逐条运行过程。

## 目标
- 让任务详情页看到每条数据的运行状态与结果
- 让任务执行过程对单条失败可追踪
- 明确任务提示词拼接与媒体处理策略

## 非目标
- 不改造普通视频为抽帧评测
- 不在任务列表页展示逐条数据状态
- 不引入消息队列或独立后台任务系统

## 决策
- 在 `task_results` 中新增逐条状态字段，例如 `pending / running / completed / failed`
- 在任务启动前，为当前数据集的每条数据预创建一条 `task_results` 记录，便于前端稳定展示全量进度
- 单条数据执行时更新对应结果记录的状态、模型输出、评分、失败原因与完成时间
- 任务提示词沿用任务级 `prompt` 优先、评测集 `annotation_prompt` 兜底，再与 `custom_tags` 拼接的策略
- 媒体处理保持现状：图片直传、普通视频直传、GIF 抽帧

## 权衡
- 预创建结果记录会增加一次批量写入，但能显著简化前端逐条进度展示
- 复用现有轮询/刷新方式实现前端更新，简单但实时性一般

## 迁移
- 为 `task_results` 增加状态、错误信息、更新时间/完成时间字段
- 兼容旧数据：旧记录默认视为 `completed`
