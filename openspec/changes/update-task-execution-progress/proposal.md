# 变更：增强评测任务运行过程与逐条结果展示

## 为什么
当前评测任务运行链路存在三个缺口：
- 虽然后端已经会把任务级 Prompt 与评测集自定义标签组合后发给模型，但这一行为没有被规范明确约束
- 任务执行时只维护任务级状态，前端无法看到每条数据当前是否待运行、运行中、成功或失败
- 任务详情页只展示最终结果字段，缺少逐条运行状态与失败信息，难以定位异常数据

## 变更内容
- 明确任务运行时使用“任务 Prompt + 评测集自定义标签”组合作为一次模型输入提示词
- 保持图片直传、普通视频直传、GIF 抽帧的现有媒体处理策略
- 为每条评测数据增加任务执行状态，并记录失败原因与结果完成时间
- 在任务详情页的结果表中展示每条数据的运行状态、模型输出和评测结果
- 在任务运行过程中逐条更新结果状态，而不是仅在全部完成后更新任务级状态

## 影响
- 受影响规范：task-management（修改）
- 受影响代码：
  - 后端：任务运行逻辑、结果模型与结果详情接口
  - 后端：数据库 `task_results` 表结构
  - 前端：任务详情页结果表展示逐条状态、失败信息与结果字段
  - 前端：任务启动后的轮询或刷新逻辑
