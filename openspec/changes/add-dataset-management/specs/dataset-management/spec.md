## 新增需求

### 需求：创建评测集
系统必须支持创建视频类型和图片类型的评测集。

#### 场景：创建视频类型评测集
- **当** 用户填写评测集名称、描述，选择类型为"视频"
- **那么** 系统创建一个视频类型评测集，状态为"草稿"

#### 场景：创建图片类型评测集
- **当** 用户填写评测集名称、描述，选择类型为"图片"
- **那么** 系统创建一个图片类型评测集，状态为"草稿"

#### 场景：评测集名称必填
- **当** 用户未填写评测集名称就提交
- **那么** 系统提示"评测集名称不能为空"

### 需求：管理评测集
系统必须提供评测集的查看、编辑、删除功能。

#### 场景：查看评测集列表
- **当** 用户访问评测集管理页面
- **那么** 系统显示所有评测集列表，包含名称、类型、数据数量、状态

#### 场景：编辑评测集信息
- **当** 用户修改评测集名称或描述
- **那么** 系统保存修改并更新时间戳

#### 场景：删除评测集
- **当** 用户删除评测集
- **那么** 系统删除评测集及其关联的所有数据

### 需求：上传评测数据
系统必须支持本地上传评测数据，并自动存储到TOS。

#### 场景：上传视频文件
- **当** 用户上传MP4、AVI、MOV格式的视频文件
- **那么** 系统将文件存储到TOS，并记录object key到数据库

#### 场景：上传图片文件
- **当** 用户上传JPG、PNG、GIF格式的图片文件
- **那么** 系统将文件存储到TOS，并记录object key到数据库

#### 场景：文件大小限制
- **当** 用户上传超过限制的文件（视频>500MB，图片>20MB）
- **那么** 系统拒绝上传并提示文件大小超限

#### 场景：文件格式限制
- **当** 用户上传不支持的文件格式
- **那么** 系统拒绝上传并提示不支持的文件格式

#### 场景：批量上传文件
- **当** 用户同时上传多个文件
- **那么** 系统并行处理上传，显示每个文件的上传进度

### 需求：从TOS导入数据
系统必须支持从TOS对象存储导入已有数据。

#### 场景：导入TOS文件
- **当** 用户输入TOS bucket和object key前缀
- **那么** 系统扫描并列出匹配的文件供用户选择

#### 场景：确认导入
- **当** 用户选择要导入的文件并确认
- **那么** 系统将文件信息关联到当前评测集

### 需求：管理评测数据
系统必须提供评测数据的查看和删除功能。

#### 场景：查看评测数据列表
- **当** 用户进入评测集详情页
- **那么** 系统显示该评测集下所有数据，包含文件名、类型、大小、标注状态

#### 场景：删除评测数据
- **当** 用户删除评测数据
- **那么** 系统删除数据库记录和TOS上的文件

#### 场景：预览数据
- **当** 用户点击预览按钮
- **那么** 系统显示图片或播放视频
