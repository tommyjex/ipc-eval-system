## 新增需求

### 需求：人工标注
系统必须支持用户手动输入数据的ground truth（真值）。

#### 场景：单条人工标注
- **当** 用户选择一条未标注的数据，输入ground truth文本
- **那么** 系统保存标注结果，标注类型标记为"人工"，数据状态更新为"已标注"

#### 场景：批量人工标注
- **当** 用户选择多条数据，输入相同的ground truth文本
- **那么** 系统为每条数据保存相同的标注结果

#### 场景：修改标注结果
- **当** 用户修改已有标注的ground truth
- **那么** 系统更新标注结果和更新时间

#### 场景：标注格式验证
- **当** 用户输入的ground truth格式不符合要求
- **那么** 系统提示格式错误并给出正确格式示例

### 需求：大模型标注
系统必须支持调用大模型自动生成ground truth。

#### 场景：单条大模型标注
- **当** 用户对单条数据发起大模型标注请求
- **那么** 系统调用大模型API，将返回结果作为ground truth保存

#### 场景：批量大模型标注
- **当** 用户选择多条数据发起批量大模型标注
- **那么** 系统创建异步任务，逐条调用大模型API并保存结果

#### 场景：大模型标注进度
- **当** 批量大模型标注进行中
- **那么** 系统显示标注进度（已完成/总数）

#### 场景：大模型标注失败
- **当** 大模型API调用失败
- **那么** 系统记录错误信息，数据状态保持"待标注"，允许重试

#### 场景：选择大模型
- **当** 用户发起大模型标注
- **那么** 系统允许用户选择使用哪个大模型（OpenAI、Claude、通义千问等）

### 需求：标注结果管理
系统必须提供标注结果的查看、导出功能。

#### 场景：查看标注结果
- **当** 用户查看已标注数据
- **那么** 系统显示ground truth、标注类型、标注时间、标注者/模型名称

#### 场景：筛选标注状态
- **当** 用户选择筛选条件（待标注/已标注/标注失败）
- **那么** 系统显示符合条件的数据列表

#### 场景：导出标注结果
- **当** 用户导出标注结果
- **那么** 系统生成JSON格式文件，包含数据ID、文件名、ground truth

### 需求：标注质量控制
系统必须支持标注结果的审核和修正。

#### 场景：标注审核
- **当** 用户查看大模型标注结果
- **那么** 系统允许用户确认或修改标注结果

#### 场景：标注历史记录
- **当** 数据被多次标注
- **那么** 系统保留所有标注历史记录

#### 场景：标注统计
- **当** 用户查看评测集统计
- **那么** 系统显示已标注数量、待标注数量、人工标注数量、大模型标注数量
