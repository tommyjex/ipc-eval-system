# 变更：为评测任务增加可选 Prompt 字段

## 为什么
当前创建评测任务时无法为单个任务设置专属 Prompt，导致任务运行时只能依赖默认提示词或其他固定逻辑，无法按任务粒度控制评测指令。

## 变更内容
- 在创建评测任务弹窗中增加可选的 Prompt 输入字段
- 在任务数据结构中保存 Prompt 内容
- 支持在任务详情页查看并编辑 Prompt
- 任务运行调用大模型时优先使用任务级 Prompt

## 影响
- 受影响规范：task-management（修改）
- 受影响代码：
  - 前端：任务创建弹窗增加 Prompt 字段，任务详情页展示并支持编辑 Prompt
  - 前端：任务 API 类型增加 Prompt 字段
  - 后端：任务模型、Schema、创建/更新接口增加 Prompt 字段
  - 后端：任务运行逻辑改为优先使用任务级 Prompt
  - 数据库：`evaluation_tasks` 表新增 `prompt` 字段
