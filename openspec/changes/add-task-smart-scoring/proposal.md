# 变更：为评测结果增加智能评分能力

## 为什么
当前评测结果列表仅展示模型输出和评分原因，没有结构化的召回率与准确率指标，前端“智能评分”按钮也尚未接入实际能力，无法对单条评测结果进行统一、可量化的质量判断。

## 变更内容
- 为评测结果增加 `recall` 与 `accuracy` 两个结构化评分字段
- 点击任务详情页“智能评分”按钮后，批量调用 `doubao-seed-2-0-pro-260215` 对结果进行智能评分
- 大模型基于标注结果与模型输出返回结构化评分：召回率、准确率、评分理由
- 任务详情页结果表展示 `召回率`、`准确率` 和 `评分理由`

## 影响
- 受影响规范：task-management（修改）
- 受影响代码：
  - 后端：`task_results` 数据模型、Schema、智能评分接口与 Ark 调用封装
  - 后端：数据库 `task_results` 表新增评分字段
  - 前端：任务详情页“智能评分”按钮接入实际调用
  - 前端：结果表列从“评测结果”调整为“召回率 / 准确率 / 评分理由”
