## 上下文
当前任务详情页已经有“智能评分”按钮入口，但没有对应的后端评分接口。`task_results` 只有 `score` 与 `score_reason`，不足以承载用户要求的 `recall`、`accuracy` 两个结构化指标。

## 目标
- 通过任务详情页按钮触发批量智能评分
- 为每条评测结果写入 `recall`、`accuracy`、`score_reason`
- 评分模型固定为 `doubao-seed-2-0-pro-260215`

## 非目标
- 不变更评测任务主模型 `target_model`
- 不引入新的异步作业系统
- 不在任务列表页展示这些评分字段

## 决策
- 在 `task_results` 表新增 `recall`、`accuracy` 数值字段
- 后端新增任务级评分接口，对当前任务下已有结果逐条评分
- 评分输入至少包含：标注结果、模型输出、任务 Prompt、评测集自定义标签
- 评分模型返回严格 JSON，包括 `recall`、`accuracy`、`reason`
- 前端点击“智能评分”后调用该接口，并在完成后刷新结果表

## 风险
- 大模型可能返回非 JSON 或异常值，需要后端做解析与兜底校验
- 批量评分时间可能较长，前端需要展示处理中状态
