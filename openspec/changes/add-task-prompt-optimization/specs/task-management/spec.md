## 新增需求
### 需求：在任务详情页触发提示词智能优化
系统必须允许用户在任务详情页对已完成智能评分的评测任务触发提示词智能优化。

#### 场景：已完成智能评分的任务显示优化入口
- **当** 用户进入任务详情页
- **并且** 当前任务已经具备可用于分析的已评分结果
- **那么** 系统展示“提示词优化”入口

#### 场景：未完成智能评分的任务不可触发
- **当** 用户进入尚未完成智能评分的任务详情页
- **那么** 系统不得允许用户触发提示词智能优化

### 需求：基于全量已评分样本生成优化输入
系统必须基于当前任务的全量已评分样本构造提示词优化输入，并重点分析标注结果与模型输出之间的差异。

#### 场景：优化输入包含关键样本字段
- **当** 系统准备执行提示词智能优化
- **那么** 系统必须纳入当前任务下全量已评分样本的对象名称、标注结果、模型输出、召回率、精确率和评分理由

#### 场景：突出 diff 问题模式
- **当** 系统构造提示词优化输入
- **那么** 系统必须重点归纳模型输出与标注结果之间的漏检、误报、空样本误报和低分样本模式

### 需求：以两阶段链路生成问题分析与优化后提示词
系统必须使用固定优化模型通过两阶段链路为任务生成问题分析、优化策略和一版完整优化后提示词。

#### 场景：使用固定优化模型
- **当** 系统执行提示词智能优化
- **那么** 系统必须使用 `doubao-seed-2-0-pro-260215`
- **并且** 系统必须开启 thinking 模式

#### 场景：第一阶段输出问题分析
- **当** 系统执行提示词智能优化的第一阶段
- **那么** 系统必须返回结构化问题摘要
- **并且** 系统必须返回提示词优化策略

#### 场景：第二阶段输出优化后提示词
- **当** 系统执行提示词智能优化的第二阶段
- **那么** 系统必须把原始 Prompt 和第一阶段输出作为输入
- **并且** 系统必须返回一版完整优化后提示词

### 需求：支持人工微调优化后提示词
系统必须允许用户在任务详情页对优化后提示词进行人工编辑，然后再决定后续动作。

#### 场景：用户可编辑优化后提示词
- **当** 用户查看提示词优化结果
- **那么** 系统必须允许用户编辑优化后提示词文本
- **并且** 系统必须允许用户复制编辑后的提示词

### 需求：支持基于优化后提示词新建对比任务
系统必须允许用户基于优化后提示词创建新的对比任务，并自动完成评测与智能评分。

#### 场景：创建对比任务并自动运行
- **当** 用户确认使用优化后提示词进行重新验证
- **那么** 系统必须基于当前任务创建一个新的对比任务
- **并且** 新任务必须继承原任务的评测集、目标模型、模型供应商和必要配置
- **并且** 新任务必须使用用户当前确认的优化后提示词作为 Prompt
- **并且** 系统必须自动运行评测任务
- **并且** 系统必须自动发起智能评分

### 需求：支持对比优化前后的任务级效果
系统必须支持展示优化前后的任务级指标对比。

#### 场景：展示任务级指标对比
- **当** 对比任务完成评测并完成智能评分
- **那么** 系统必须支持展示原任务与对比任务的任务级指标对比
- **并且** 默认至少包含 micro recall、micro precision、macro recall、macro precision、coverage rate、empty sample pass rate 与 unscorable count

### 需求：支持将优化后提示词替换为当前任务 Prompt
系统必须允许用户显式确认后，把优化后提示词替换为当前任务 Prompt。

#### 场景：显式替换当前任务 Prompt
- **当** 用户在任务详情页点击“替换为任务 Prompt”
- **那么** 系统必须将当前编辑后的优化提示词写回当前任务 Prompt
- **并且** 系统不得在未经用户明确操作的情况下自动覆盖当前任务 Prompt
