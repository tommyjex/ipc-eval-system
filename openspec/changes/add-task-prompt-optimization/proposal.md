# 变更：为评测任务增加提示词智能优化

## 为什么
当前任务详情页已经能展示全量评测结果、智能评分结果和评分理由，但用户仍需要人工阅读大量样本 diff，自己归纳问题模式并手工修改任务 Prompt，效率低且难以复用。

对于已经完成智能评分的任务，系统已经具备提示词优化所需的核心输入：对象名称、标注结果、模型输出、召回率、精确率与评分理由。当前虽然已具备“问题诊断 + 优化建议 + 新提示词”的基础能力，但仍缺少真正可落地的后续动作闭环：
- 用户无法在系统内继续微调优化后提示词
- 无法直接基于优化后提示词新建对比任务并重跑评测
- 无法系统化对比优化前后的任务级指标变化
- 无法一键把优化后提示词替换为当前任务 Prompt

## 变更内容
- 在任务详情页为“已完成智能评分”的任务增加“提示词优化”入口
- 以任务下全量已评分样本作为输入，重点分析 `标注结果 vs 模型输出` 的差异并归纳高频问题模式
- 使用固定优化模型 `doubao-seed-2-0-pro-260215`，并开启 thinking 模式执行两阶段优化链路
- 第一阶段输出问题分析与优化策略
- 第二阶段基于原始 Prompt 和第一阶段结论输出一版完整优化后提示词
- 输出结构化优化结果，包括问题摘要、典型 diff、优化策略和一版完整优化后提示词
- 支持用户在任务详情页对优化后提示词进行人工微调
- 支持基于优化后提示词新建“对比任务”，并自动运行评测与智能评分
- 支持从任务级指标维度对比优化前后的效果
- 支持用户显式确认后将优化后提示词替换为当前任务 Prompt

## 影响
- 受影响规范：`task-management`
- 受影响代码：
  - 后端：任务详情相关 API、提示词优化调用链路、优化结果持久化与对比任务创建
  - 前端：任务详情页入口、优化结果展示、人工微调、对比结果展示、替换当前 Prompt 交互
  - 模型调用：新增固定优化模型配置与 thinking 模式调用
