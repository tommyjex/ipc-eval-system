## 新增需求
### 需求：按模型供应商执行评测任务
系统必须根据评测任务配置的模型供应商调用对应平台的模型接口完成评测，而不是固定使用单一供应商。

#### 场景：火山引擎任务继续走方舟
- **当** 评测任务的 `model_provider` 为 `volcengine`
- **那么** 系统继续调用火山引擎方舟接口执行评测

#### 场景：阿里千问任务走阿里百炼
- **当** 评测任务的 `model_provider` 为 `aliyun`
- **那么** 系统调用阿里百炼 DashScope 接口，并使用任务指定的千问模型执行评测

### 需求：阿里千问支持多种输入类型
系统必须在阿里千问评测任务中支持文本、图片和视频输入。

#### 场景：文本输入评测
- **当** 系统执行仅文本输入的阿里千问评测任务
- **那么** 系统使用阿里百炼文本生成接口完成评测

#### 场景：图片输入评测
- **当** 系统执行图片输入的阿里千问评测任务
- **那么** 系统使用阿里百炼多模态接口完成评测

#### 场景：视频输入评测
- **当** 系统执行视频输入的阿里千问评测任务
- **那么** 系统使用阿里百炼多模态接口完成评测

#### 场景：GIF 输入评测
- **当** 系统执行 GIF 输入的阿里千问评测任务
- **那么** 系统继续采用抽帧策略，并将抽出的多帧图片发送给阿里千问多模态接口

### 需求：供应商失败时返回清晰错误
系统必须在供应商调用失败或模型不兼容时，将失败原因记录到任务结果中。

#### 场景：阿里百炼调用失败
- **当** 阿里百炼接口返回错误或请求失败
- **那么** 系统将该条任务结果标记为失败并记录错误原因
