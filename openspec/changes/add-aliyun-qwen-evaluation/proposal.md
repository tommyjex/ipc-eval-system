# 变更：支持阿里云千问系列模型评测

## 为什么
当前评测任务前端已经允许选择 `阿里千问` 作为模型供应商，但后端执行评测时仍然固定调用火山引擎方舟接口，导致选择阿里千问时无法真正使用阿里百炼平台的模型完成评测。

为了支持更多模型供应商并满足用户对千问系列模型的测评需求，系统需要在任务运行阶段根据模型供应商动态切换到阿里百炼 DashScope 接口，并支持文本、图片和视频输入。

## 变更内容
- 为评测任务新增阿里云百炼 DashScope 调用链路
- 当 `model_provider = aliyun` 时，任务运行改为调用阿里千问模型
- 支持以下输入类型：
  - 文本输入
  - 图片输入
  - 视频输入
- 复用现有 GIF 抽帧策略，将 GIF 作为多帧图片输入给阿里千问多模态模型
- 新增阿里百炼配置项，例如 API Key 与基础地址
- 前端任务创建页补充适合千问的模型选项与文案说明

## 影响
- 受影响规范：task-management（修改）
- 受影响代码：
  - 后端：任务执行分流逻辑、模型客户端抽象、阿里百炼调用实现、配置项
  - 前端：任务创建页的阿里千问模型选项
  - 文档：环境变量与配置说明
