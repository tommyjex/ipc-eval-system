## 新增需求

### 需求：前端项目结构
系统必须提供标准化的前端项目结构，支持 React + TypeScript 技术栈。

#### 场景：创建前端项目
- **当** 开发者初始化前端项目
- **那么** 应生成包含 components、pages、hooks、utils、api 目录的结构

#### 场景：配置代码规范
- **当** 开发者运行 lint 命令
- **那么** 应按照 ESLint 和 Prettier 规则检查代码

### 需求：后端项目结构
系统必须提供标准化的后端项目结构，支持 Node.js 技术栈。

#### 场景：创建后端项目
- **当** 开发者初始化后端项目
- **那么** 应生成包含 routes、controllers、services、models、middleware 目录的结构

#### 场景：API 服务启动
- **当** 开发者启动后端服务
- **那么** 服务应监听配置的端口并返回健康检查响应

### 需求：环境变量管理
系统必须支持通过环境变量配置运行参数。

#### 场景：加载环境变量
- **当** 应用启动时
- **那么** 应从 .env 文件加载环境变量

#### 场景：环境变量验证
- **当** 缺少必需的环境变量
- **那么** 应用应报错并拒绝启动

### 需求：共享配置
系统必须提供跨项目的共享配置，确保代码风格一致性。

#### 场景：共享 TypeScript 配置
- **当** 前后端项目编译时
- **那么** 应使用统一的 TypeScript 配置基础

#### 场景：共享代码风格
- **当** 运行代码格式化
- **那么** 前后端项目应使用统一的格式化规则
