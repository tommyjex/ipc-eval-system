# 变更：支持评分标准模板管理与套用

## 为什么
当前评测任务的评分标准只能在创建任务或编辑任务时手工输入，无法复用已有评分标准，也无法按业务场景统一维护，导致相同场景下重复录入、内容不一致、维护成本高。

为了提高评分标准复用效率，系统需要支持可维护的评分标准模板，并在任务创建和任务详情编辑时按评测集业务场景选择模板直接套用。

## 变更内容
- 新增评分标准模板数据模型与 CRUD 接口
- 模板按业务场景分类，并复用评测集已有的 `scene` 分类
- 在创建任务时支持按评测集场景选择模板并覆盖评分标准
- 在编辑任务详情时支持按评测集场景选择模板并覆盖评分标准
- 提供最小可用的模板管理界面，支持创建、编辑、删除模板

## 影响
- 受影响规范：task-management（修改）
- 受影响代码：
  - 后端：评分模板模型、接口、应用启动建表
  - 前端：任务创建页、任务详情编辑页、模板管理弹窗
