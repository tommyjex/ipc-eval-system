# [OPEN] GIF Key Migration

## 背景
- 目标：修复 `01-摄像头光线自变.gif` 的 `tos_key` 迁移问题。
- 范围：先收集证据，再决定是否更新数据库记录。

## 当前假设
- 假设 1：该对象在 TOS 中存在新 key，但文件名发生了编号或局部文案变化。
- 假设 2：该对象在数据库中存在另一条记录，指向了新的有效 key。
- 假设 3：该对象已从 TOS 删除，当前并不存在可迁移目标。
- 假设 4：该对象迁移到了不同目录前缀，原模糊搜索范围不足。
- 假设 5：存在内容等价但文件名差异较大的替代对象，需要借助邻近样本和尺寸信息识别。

## 计划
- 扫描数据库与 TOS 中相近文件名、相邻编号和近似关键词。
- 比对对象大小、目录前缀和记录分布，确定是否存在可用迁移目标。
- 仅在找到明确有效目标后再执行数据库修复。

## 结果
- 数据库与 TOS 目录级对账结果：
  - `DB_ONLY_COUNT = 2`
  - `DB_ONLY = 01-大哥骑电瓶车.gif`
  - `DB_ONLY = 01-摄像头光线自变.gif`
  - `TOS_ONLY_COUNT = 0`
- 已执行第一条记录修复后，TOS 目录内不存在任何额外的 `.gif` 对象可作为 `01-摄像头光线自变.gif` 的迁移目标。
- 关键词模糊搜索未命中任何包含 `摄像头` / `光线` / `自变` 的 `.gif` 对象。

## 假设结论
- 假设 1：否决。未找到文件名变化后的现存 GIF key。
- 假设 2：否决。数据库中不存在另一条可对应的新记录。
- 假设 3：当前最成立。该对象更像是已从 TOS 删除，暂无可迁移目标。
- 假设 4：暂未支持。扩大到当前可访问目录级对象后仍未发现候选。
- 假设 5：暂未支持。没有发现内容等价但文件名差异较大的现存 GIF 对象。
